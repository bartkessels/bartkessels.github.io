CMake is the de facto standard for building C++ projects. In this article, we'll cover the basics you need to get started with CMake for your C++ projects.

## What is CMake?

CMake is a cross-platform build system generator. It doesn't build your project directly; instead, it generates native build files for your platform (Makefiles on Linux, Visual Studio projects on Windows, etc.).

## Project Structure

A typical CMake project structure looks like this:

```
my_project/
├── CMakeLists.txt
├── src/
│   ├── main.cpp
│   └── utils.cpp
├── include/
│   └── utils.h
└── build/
```

## Basic CMakeLists.txt

Here's a minimal CMakeLists.txt to get started:

```cmake
# CMakeLists.txt
cmake_minimum_required(VERSION 3.20)

# Project name and version
project(MyProject VERSION 1.0.0 LANGUAGES CXX)

# Set C++ standard
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# Add executable
add_executable(my_app
    src/main.cpp
    src/utils.cpp
)

# Include directories
target_include_directories(my_app PRIVATE include)
```

## Building Your Project

To build with CMake, use these commands:

```bash
# Create and enter build directory
mkdir build && cd build

# Generate build files
cmake ..

# Build the project
cmake --build .
```

## Adding Libraries

CMake makes it easy to add external libraries:

```cmake
# Find and link a library
find_package(fmt REQUIRED)
target_link_libraries(my_app PRIVATE fmt::fmt)

# Or create your own library
add_library(my_lib STATIC
    src/my_lib.cpp
)

target_link_libraries(my_app PRIVATE my_lib)
```

## Conclusion

CMake is a powerful tool that, once mastered, makes C++ project management much more manageable. Start with the basics and gradually explore its more advanced features.
