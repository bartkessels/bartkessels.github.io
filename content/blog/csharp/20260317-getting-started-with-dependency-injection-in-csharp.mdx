Dependency Injection (DI) is a fundamental concept in modern C# development. It helps us write loosely coupled, testable, and maintainable code. In this article, we'll explore how to implement DI in your .NET applications.

## What is Dependency Injection?

Dependency Injection is a design pattern where an object receives its dependencies from external sources rather than creating them itself. This inverts the control of object creation, hence the term "Inversion of Control" (IoC).

Consider this example without DI:

```csharp
// Without Dependency Injection
public class OrderService
{
    private readonly EmailService _emailService;
    
    public OrderService()
    {
        // Tight coupling - OrderService creates its own dependency
        _emailService = new EmailService();
    }
    
    public void PlaceOrder(Order order)
    {
        // Process order...
        _emailService.SendConfirmation(order.CustomerEmail);
    }
}
```

Now, let's refactor this using DI:

```csharp
// With Dependency Injection
public class OrderService
{
    private readonly IEmailService _emailService;
    
    public OrderService(IEmailService emailService)
    {
        // Dependency is injected through the constructor
        _emailService = emailService;
    }
    
    public void PlaceOrder(Order order)
    {
        // Process order...
        _emailService.SendConfirmation(order.CustomerEmail);
    }
}
```

## Setting Up DI in .NET

.NET has built-in support for dependency injection. Here's how to set it up in a typical ASP.NET Core application:

```csharp
// Program.cs
var builder = WebApplication.CreateBuilder(args);

// Register services
builder.Services.AddScoped<IEmailService, EmailService>();
builder.Services.AddScoped<IOrderService, OrderService>();
builder.Services.AddScoped<IOrderRepository, OrderRepository>();

var app = builder.Build();
```

## Service Lifetimes

Understanding service lifetimes is crucial for proper DI implementation:

- **Transient**: A new instance is created each time the service is requested
- **Scoped**: One instance per request/scope
- **Singleton**: One instance for the entire application lifetime

```csharp
// Different service lifetimes
builder.Services.AddTransient<ITransientService, TransientService>();
builder.Services.AddScoped<IScopedService, ScopedService>();
builder.Services.AddSingleton<ISingletonService, SingletonService>();
```

## Conclusion

Dependency Injection is a powerful pattern that makes your code more modular, testable, and maintainable. By leveraging .NET's built-in DI container, you can easily manage your application's dependencies without additional libraries.
